# Agent Orchestration Platforms

Visual, managed, and low-code platforms for building, deploying, and coordinating AI agent workflows — lowering the barrier from code-first agent frameworks.

## What This Concept Covers

Not everyone wants to write Python to build agents. These platforms provide:

- **Visual builders** — drag-and-drop interfaces for agent workflow design
- **Managed orchestration** — platforms that handle agent lifecycle, scaling, and coordination
- **Low-code agent creation** — building agents through configuration rather than code
- **Production deployment** — turning agent prototypes into reliable services

## Key Approaches

**Visual workflow builder (Flowise):**
- Drag-and-drop canvas for connecting LLM nodes
- No code required for basic agent workflows
- Good for prototyping and non-developers
- Best for: teams without deep coding skills

**Claude-focused orchestration (Ruflo):**
- Purpose-built for Claude agent coordination
- Multi-agent swarm patterns
- Best for: Claude-centric workflows

**Autonomous business operations (Paperclip):**
- "Zero-human company" vision
- Agents handle business processes end-to-end
- Best for: automating entire business workflows

**Managed agent teams (Multica):**
- Treats agents as team members, not tools
- Persistent context and memory across sessions
- Best for: long-running agent deployments

## When to Use

- **Flowise** — visual prototyping, non-developer teams
- **Ruflo** — Claude-specific multi-agent workflows
- **Paperclip** — full business process automation
- **Multica** — agents as persistent team members
